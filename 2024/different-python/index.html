Missing PIP Module Error

Posted on 2024-01-23

If you are constantly facing the missing pip module error even after you’ve installed it multiple times, it is very likely that you are running a different python instance than the one you are installing the pip modules to.

Solution

In VS Code, press Ctrl + Shift + P, type and select Python: Select Interpreter. You will likely have more than one pythons (maybe in different versions) in the list that pops up. Go through the list and try each one until you can successfully run the code.

Root Cause

The root cause of this issue is a repeated installation of Python, likely because Anaconda installed one or VS Code Python extension installed one, and you installed another one manually, causing the terminal (cmd/powershell on windows) to use the Python you installed manually, and VS Code still using the Anaconda one or the Python extension auto-installed one.

The lent booster page linked on Moodle can help you debug this as well, but you might find the above solution more direct and more efficient.

Another cause might just be regular venv user who have accidentally messed around too much. Don’t think that’s very common.
